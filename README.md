Most of my work lives at one intersection: **AI agents that actually finish the job.** The model is
rarely the bottleneck — context hygiene, token cost, and *verification* are. So I build the unglamorous
plumbing that turns a generic agent into a reliable operator, and I keep it free, local, and
dependency-light so anyone can run it.

### 🚀 Featured projects

| Project | What it does |
|---|---|
| **[handler](https://github.com/za3ter123/handler)** | Autonomous multi-agent orchestration harness for Claude Code — a verification-first plan→dispatch→verify→ship loop with writer≠reviewer panels, difficulty-aware model routing, and a self-learning rule engine. |
| **[claude-skill-rag](https://github.com/za3ter123/claude-skill-rag)** | Lazy skill loading that makes an unlimited number of agent skills usable at **near-zero per-turn context cost** (measured ~6.9k → ~0.6k tokens/turn). |

### 🧭 How I build

- **Verification-first.** Done means *proven* — tests run, output shown. The agent that writes code never reviews it.
- **Local & free by default.** Offline where possible, no keys you don't need, no per-use bill.
- **Token-aware.** Context is the budget. I measure it and cut waste instead of guessing.
- **Small, readable, MIT.** Few dependencies, clear docs, take it and run.
